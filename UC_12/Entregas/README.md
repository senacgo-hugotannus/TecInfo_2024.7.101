# Instruções para Entrega de Exercícios da UC 12

> [!IMPORTANT]
> Antes de executar qualquer um dos comandos abaixo, certifique-se de estar na pasta corretamente associada ao repositório.

> [!IMPORTANT]
> Certifique-se de deixar a pasta organizada, de forma a facilitar a identificação correta das entregas.

> [!TIP]
> Para _clonar_ (_"baixar"_) o projeto, basta abrir o terminal na pasta "Projetos" e executar o seguinte comando:
>
> `git clone https://github.com/senacgo-hugotannus/TecInfo_2024.7.101.git`

> [!TIP]
> Lembre-se de que você sempre poderá criar o seu próprio arquivo `README.md` em sua pasta, para deixar as suas entregas ainda mais organizadas, além de ser o local perfeito para registrar quaisquer esclarecimentos que se fizerem necessários.

1. Execute o comando `git pull`
1. Crie uma pasta com o seu nome ou a identificação do seu grupo dentro da pasta adequada.
    - **Exemplo:** para entregar atividades referentes a _"registros e dicionários"_, um aluno que seja do grupo **N** deverá criar uma nova pasta `Grupo_N` dentro da pasta `Registros_e_Dicionarios` reservada para a entrega de tal atividade e, então, adicionar os arquivos referente às respostas das atividades.
2. Crie uma nova _branch_ para a entrega a ser realizada, identificando tanto a tarefa quanto o grupo em questão.
    - **Exemplo:** para criar uma nova _branch_ com o nome `registros_grupo_n`, basta executar o comando `git checkout -b registros_grupo_n`.
2. Adicione os arquivos e subpastas referentes à entrega, faça os _commits_ e sincronize a nova _branch_. Certifique-se de que a nova _branch_ apareça no repositório do Git Hub.
3. Crie um novo _Pull Request_ no GitHub, à partir da _branch_ criada, apontando para a branch principal (`main`).
4. Acompanhe o _Pull Request_ criado para possíveis revisões de código solicitadas pelo Instrutor.
